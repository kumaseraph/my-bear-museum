# 🐻 熊熊配送 Playbook

> 建立日期：2026-06-09
> 更新者：KumaBao
> 版本：v1.0

---

## 📌 核心原則

1. **每日限量**：每天固定 6 張圖片
2. **風格輪流**：39 種風格不重複循環
3. **工具分工**：ComfyUI 和 MiniMax 各司其職
4. **詞彙驅動**：熊熊命名來自詞彙池
5. **準時上架**：每日早上完成所有配送

## 📝 熊熊命名系統

### 詞彙參照流程

```
┌─────────────────────────────────────────────────────────────┐
│  daily-words-pool.json                                      │
│  （每日詞彙池，500+ 詞）                                     │
│  例如：暖心窩、甜蜜果、幸福糖、陽光籽、希望苗...              │
└────────────────────┬──────────────────────────────────────┘
                     │ daily-vocab-update.py（每日）
                     ↓
┌─────────────────────────────────────────────────────────────┐
│  world-building.json                                        │
│  （世界觀詞庫，每日新增 10 詞）                              │
│  categories: 地名、自然、情感、寶石、星空、花草...           │
└────────────────────┬──────────────────────────────────────┘
                     │ 熊熊命名時取用
                     ↓
┌─────────────────────────────────────────────────────────────┐
│  bear-naming.json                                          │
│  （命名字典，500+ 組合）                                    │
│  prefix（來自 world-building）+ core + suffix                │
└─────────────────────────────────────────────────────────────┘
```

### 具體參照

| 檔案 | 用途 | 與命名系統的關係 |
|------|------|-----------------|
| `daily-words-pool.json` | 每日詞彙池 | → 餵入 world-building.json |
| `world-building.json` | 世界觀詞庫 | → 提供前綴詞彙 |
| `bear-naming.json` | 命名字典 | 組合前綴+核心+後綴 |

### 命名組合規則

```
熊熊名稱 = 前綴（prefix）+ 核心（core）+ 後綴（suffix）
格式：前綴 + 核心 + 熊/稱號

範例：晨曦熊、流雲熊、月影熊、極光熊、暖風熊、星願熊、海霧熊、雪絨熊
      晨曦旅人、暖陽精靈、月影仙子、星光守護者、銀河收藏家
```

### 前綴詞來源

`bear-naming.json` 中的 **prefix** 詞彙來自 `world-building.json`，包含：

| 類別 | 詞彙範例 |
|------|----------|
| 時間/氛圍 | 晨曦、暖陽、暮色、曙光、暮靄、晨風、暖風 |
| 自然元素 | 翠綠、琥珀、白雲、彩霞、晨霧、晚霞、微風 |
| 色彩 | 金色、紫色、粉色、珊瑚、蔚藍、蜜糖、奶油 |
| 珍寶 | 珍珠、水晶、琉璃、雪花、露珠、極光、彩虹 |
| 風景 | 雲端、森林、海洋、湖畔、星空、月光、清泉 |

### 後綴稱號（固定）

```
熊、旅人、精靈、守護者、收藏家、仙子、使者、
```

## 🌐 發布網頁流程

每次更新內容（修改 HTML/CSS、添加新熊熊等）後：

```
1️⃣ 修改 index.html（內容 + 版本號）
        ↓
2️⃣ Git 提交 + 推送
   git add -A && git commit -m "vX.Y - 描述" && git push
        ↓
3️⃣ 部署到 Cloudflare Pages
   npx wrangler pages deploy . --project-name kumaweb --branch main --no-install-skills --commit-dirty=true
        ↓
4️⃣ 驗證部署成功
```

### 版本號更新位置（三處）

| 位置 | 說明 |
|------|------|
| `<title>` 標籤 | 行 ~39 |
| footer 版本文字 | 行 ~2012 |
| version history 註解 | 行 ~28 |

### 重要原則

| 原則 | 說明 |
|------|------|
| **版本號先更新** | 先更新版本號再部署，否則需要兩次 deploy |
| **三處要一致** | `<title>`、footer 版本、version history 註解 |
| **先 push 再部署** | 否則會部署舊版本 |

### 驗證指令

```bash
# 確認版本號

# 確認 CF Pages 正常
```
